软件测试的艺术：02

第三章 代码检查、走查与评审

  • 不依赖计算机，从技术上探讨“如何有效的测试”

人工测试方法

  • 测试阶段：在程序开始编码之后、基于计算机的测试开始之前

3.1 检查与走查

优点：

  • 对过去桌面检查过程的改进，因为测试的主要工作是由其他人，而不是软件编写者本人来完成
  • 一旦发现错误，通常就能在代码中对其进行精确定位，这就降低了调试（错误修正）的成本
  • 通常可以发现成批的错误，这样错误就可以一同得到修正。

3.2 代码检查 Code Inspections

一个检查代码小组的组成：

  • 协调人（相当于质量控制工程师）
  • 该程序的编码人员
  • 该程序的设计人员
  • 测试专家

代码检查的过程：

  • 由程序编码人员逐条语句讲述程序的逻辑结构
  • 对着历来常见的编码错误列表分析程序

3.3 用于代码检查的错误列表

3.4 代码走查

同样是小组会议的形式，

不同之处在于：参与者会准备一些测试用例按照程序的逻辑结构在人们脑中推演

3.5 桌面检查 Desk Checking

  • 由单人进行的代码检查或代码走查：由一个人阅读程序，对照错误列表检查程序，对程序推演测试数据
  • 效率低

3.6 同行评分

  • 同行评分是一种依据程序整体质量，可维护性、可扩展性、易用性和清晰性对

    匿名程序进行评价的技术
